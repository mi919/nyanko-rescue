## 3. BGM

| No. | ID | 使用場面 | テンポ | 調 | イメージ |
|-----|-----|---------|--------|-----|---------|
| 1 | `bgm_title` | タイトル画面 | ゆったり 80BPM | メジャー | 穏やかで明るい、猫のいる庭の午後 |
| 2 | `bgm_game` | ゲーム画面（通常） | 中程度 110BPM | メジャー | 軽い緊張感、探索のワクワク |
| 3 | `bgm_game_tense` | ゲーム画面（ライフ1） | やや速い 130BPM | マイナー | 緊迫感、でも暗すぎない |
| 4 | `bgm_encounter` | 仲間との出会い画面 | ゆったり 90BPM | メジャー | 感動的、温かい |

### BGM切替ルール

```
タイトル画面表示 → bgm_title 再生
「はじめる」押下 → bgm_title フェードアウト(500ms) → bgm_game 再生
ライフが1になる → bgm_game → bgm_game_tense にクロスフェード(1000ms)
ライフが回復して2以上 → bgm_game_tense → bgm_game にクロスフェード
ステージクリア → BGM停止 → se_stage_clear
ゲームオーバー → BGM停止 → se_game_over
出会い画面表示 → bgm_encounter 再生
タイトルに戻る → bgm_encounter フェードアウト → bgm_title
```

## 4. 音響デザインガイドライン

### 4-1. トーン

- **温かみ重視**: 金属的・電子的な音よりも、木琴・鉄琴・オルゴール系
- **怖すぎない**: 犬の鳴き声や衝撃音はコミカル寄り
- **達成感**: クリア系の音は華やかに、プレイヤーを祝福する
- **猫的**: 可能であれば猫の鳴き声（ニャー）を保護音やBGMに組み込む

### 4-2. ボリュームバランス

| カテゴリ | 相対ボリューム |
|---------|-------------|
| BGM | 0.3（背景として控えめ） |
| SE（通常） | 0.6 |
| SE（重要：クリア/ゲームオーバー） | 0.8 |
| SE（UI） | 0.4（控えめ） |

### 4-3. ファイル形式

| 項目 | 推奨 |
|------|------|
| 形式 | MP3 or OGG（ブラウザ互換性） |
| サンプルレート | 44.1kHz |
| ビットレート | SE: 128kbps / BGM: 192kbps |
| チャンネル | モノラル（SE）/ ステレオ（BGM） |

### 4-4. 実装方針

Web Audio API or Howler.js の使用を推奨。

```javascript
// 基本パターン
const playSound = (id, volume = 0.6) => {
  const audio = new Audio(`/assets/sounds/${id}.mp3`);
  audio.volume = volume;
  audio.play().catch(() => {}); // ユーザー操作前のautoplay制限対策
};
```

モバイルでは最初のタップ時にAudioContextを初期化する必要がある。
