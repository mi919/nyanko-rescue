# 🖼 ステージ背景画像 仕様書

**バージョン:** 1.0
**最終更新:** 2026/05/03

---

## 1. 概要

ゲームメインの 4 ステージ（にわ / こうえん / しょうてんがい / はいきょ）に、それぞれ専用の背景イラストを追加する。
従来は `STAGES[].bg` に単色 (`#e8f5e9` など) を指定して `linear-gradient` を敷くだけだったが、本仕様では **ステージごとの世界観イラスト**を用意し、グラスモーフィズム化された UI の背後に配置する。

### 1-1. 目的

- ステージごとの個性・進行感を視覚的に演出する
- グラスUI（`backdrop-filter` で透ける）の魅力を最大化するための「背景の絵柄」を提供する
- 周回プレイで飽きない情景の変化を作る

### 1-2. 既存仕様との関係

| ファイル | 影響 |
|---------|------|
| `docs/asset-spec.md` | §4「メインビジュアル」(タイトル背景) と並ぶ位置づけ。本仕様書はゲーム画面背景を担当 |
| `docs/ui-spec.md` §3 ゲーム画面 | 背景レイヤーが追加されることを明記する形で本仕様書から参照する |
| `src/constants/stages.ts` | 単色 `bg` は **フォールバック色**として残す。画像URLを別フィールドで持つ |
| `src/screens/GameScreen.tsx` | 背景アンビエント（流れるブロブ）の上または下にイメージレイヤーを敷く |

## 2. 全体方針

### 2-1. デザインテイスト

| 項目 | 仕様 |
|------|------|
| テイスト | やさしいアニメ調イラスト（タイトル背景 `main.png` と同系統） |
| 色調 | パステル基調、彩度やや抑えめ |
| アウトライン | 太め（1〜2px相当）、にじみのある線 |
| 雰囲気 | 各ステージの時間帯・場所感が一目でわかる |
| 主役 | **猫が住んでいそうな世界感**。ただし猫・犬は描かない（盤面に描かれているため） |

### 2-2. レイアウト原則（重要）

中央の盤面とステータスUIを **絶対に妨げない**こと。

```
画面中央（縦60〜70%）の領域 = UIゾーン
  → 細部の描き込みを避け、彩度を落とした「霞んだ」描写にする
画面外周（上下左右）= 背景ゾーン
  → ここに主要モチーフを配置（地面・空・建物など）
```

具体的には:

- **中央 30〜70%** の領域は、平均明度 ≥ 90% 相当の単色〜淡いグラデにする（盤面の視認性確保）
- 主要モチーフ（建物・木・廃墟など）は **上端 20%** または **下端 20%** に集中させる
- 細かいディテール（葉脈・看板の文字など）は中央に描かない
- **中央付近に強いコントラストを置かない**（黒い影・濃い線などNG）

### 2-3. グラスUIとの相性

UI 要素は `backdrop-filter: blur(14〜18px) saturate(140〜160%)` でぼかし＋彩度UPする。

- 背景がのっぺりすぎるとガラスが透けてもつまらない → ある程度の **色の変化**が欲しい
- 逆に細かい線が密集していると blur で潰れて汚くなる → **大きめのフォルム**で構成する
- パステル系の暖色×寒色の組み合わせがガラス越しに最も美しく見える

### 2-4. 周回プレイ配慮

- 同じステージを何度も見ても疲れないよう、**情報量を抑える**（漫画よりも壁紙寄り）
- 複雑なストーリー要素（人物・ドラマ的演出）を入れない
- 静的な絵に見えるが、`ambientFloat` 等のCSSで部分的に動かす想定

## 3. ファイル仕様

### 3-1. 共通仕様

| 項目 | 値 |
|------|-----|
| ディレクトリ | `src/assets/stages/` |
| ファイル名 | `stage_<key>.png`（key: `niwa` / `kouen` / `shouten` / `haikyo`） |
| 形式 | PNG または JPEG（写真寄りなら JPEG q=80） |
| 解像度（基準） | **1200×1800px**（縦長 2:3）|
| 解像度（最大） | 1500×2250px（高DPI 端末用）|
| ファイルサイズ目標 | 1ファイル 200〜350KB |
| カラースペース | sRGB |
| 透過 | **不要**（最下層レイヤーとして敷く） |
| 命名 | snake_case + ステージキー（後述 §3-2） |

### 3-2. キー命名対応表

| ステージ表示名 | キー（ASCII） | ファイル名 |
|---------------|--------------|----------|
| にわ | `niwa` | `stage_niwa.png` |
| こうえん | `kouen` | `stage_kouen.png` |
| しょうてんがい | `shouten` | `stage_shouten.png` |
| はいきょ | `haikyo` | `stage_haikyo.png` |

### 3-3. 解像度の設計理由

- iPhone 縦持ち（430×932 @3x = 1290×2796）でも縦方向に余裕が出るよう **2:3 比率**を採用
- 横持ちや iPad には `object-fit: cover` で対応
- 1200×1800 = 約 2.16MP。PNG-8 量子化（256色）で 200KB 前後に収まる

### 3-4. 量子化・圧縮

- PNG: 256色パレット + Floyd-Steinberg ディザリング（既存スプライトと同方式）
- JPEG: quality 80、プログレッシブ
- WebP は採用しない（互換性優先、PNG/JPEG 統一）

## 4. 表示方法

### 4-1. 配置レイヤー

`GameScreen` の z-index 構成:

```
z-index:  0 → ステージ背景画像（本仕様）   ← 新設
z-index:  0 → 背景アンビエントブロブ（既存、画像の上に重ねる）
z-index:  1 → UIラッパー（盤面・ステータス・ゲージ）
z-index: 150 → トースト
z-index: 200 → モーダル
z-index: 270 → クリア/ゲームオーバーパネル
```

### 4-2. CSS実装案

```css
.stage-bg {
  position: absolute;
  inset: 0;
  background-image: url(<stage_niwa.png>);
  background-size: cover;
  background-position: center top;
  background-repeat: no-repeat;
  background-color: var(--stage-fallback);  /* 画像読込失敗時の単色フォールバック */
  z-index: 0;
  pointer-events: none;
  /* ステージ間の遷移をなめらかに */
  transition: opacity 0.5s ease;
}
```

### 4-3. 暗くしすぎないオーバーレイ

UIゾーンの視認性確保のため、画像の上に **薄い白オーバーレイ**を敷く:

```css
.stage-bg::after {
  content: "";
  position: absolute; inset: 0;
  background: linear-gradient(
    180deg,
    rgba(255,255,255,0) 0%,
    rgba(255,255,255,0.35) 30%,
    rgba(255,255,255,0.45) 70%,
    rgba(255,255,255,0) 100%
  );
}
```

中央が最も明るく、上下端は素のイラストが見える構造。

### 4-4. 既存単色の扱い

`STAGES[].bg` は削除せず、**フォールバック色**として保持する:

```ts
{ name: "にわ", bg: "#e8f5e9", bgImage: "stage_niwa.png", ... }
```

画像読み込み中・失敗時にはこの単色が見える。

### 4-5. アセット import パターン

既存の `main.png` と同様、Vite の静的 import で:

```ts
import niwaUrl    from "../assets/stages/stage_niwa.png";
import kouenUrl   from "../assets/stages/stage_kouen.png";
import shoutenUrl from "../assets/stages/stage_shouten.png";
import haikyoUrl  from "../assets/stages/stage_haikyo.png";

export const STAGE_BG_URLS: Record<string, string> = {
  niwa: niwaUrl, kouen: kouenUrl, shouten: shoutenUrl, haikyo: haikyoUrl,
};
```

## 5. ステージ別 詳細仕様

### 5-1. にわ（stage_niwa.png）

| 項目 | 値 |
|------|-----|
| 時間帯 | 昼下がり（明るい順光） |
| 場所 | 一般家庭の庭・縁側 |
| キー絵文字 | 🌿 |
| フォールバック色 | `#e8f5e9`（薄緑） |

**シーン構成:**
- 上端 20%: 青空＋ふわふわ雲（白〜薄い水色 `#bbdefb`）、太陽光の反射
- 中央 60%: **芝生の地面**（パステルグリーン `#a5d6a7`〜`#c8e6c9`）、ぼやけた花壇 (淡いピンク `#f8bbd0`、黄色 `#fff59d`)
- 下端 20%: 縁側の木板・植木鉢のシルエット、踏み石、シロツメクサ

**配色パレット（参考）:**
```
空: #bbdefb / #e3f2fd
芝生: #a5d6a7 / #c8e6c9 / #dcedc8
花: #f8bbd0 / #fff59d / #ffccbc
木材: #d7ccc8 / #bcaaa4
影: #c5cae9（青寄りの淡影、決して黒影は使わない）
```

**避けること:**
- 動物（猫・犬・鳥が中央付近にいる）→ 盤面と被る
- 中央に大きな花柄 → blur で潰れて汚くなる
- 真っ赤な花など彩度の高いオブジェクト

**雰囲気キーワード:** のどか・初春・午後3時・最初のステージにふさわしい安心感

---

### 5-2. こうえん（stage_kouen.png）

| 項目 | 値 |
|------|-----|
| 時間帯 | 夕方の少し手前（黄金時間） |
| 場所 | 街中の公園 |
| キー絵文字 | 🌳 |
| フォールバック色 | `#e3f2fd`（薄青） |

**シーン構成:**
- 上端 25%: 夕焼け前のグラデ空（薄水色 → 淡いオレンジ）、雲、シルエットの飛行機
- 中央 50%: ぼかし強めの **芝生・舗装路**、左右奥にぼやけたベンチや街灯
- 下端 25%: 公園の入口アーチ・大きな木のシルエット（左下・右下）、噴水、ボール

**配色パレット（参考）:**
```
空: #b3e5fc → #ffe0b2 のグラデ
芝生: #c5e1a5 / #aed581
木のシルエット: #689f38（彩度低め）
舗装路: #cfd8dc / #b0bec5
ベンチ: #a1887f / #d7ccc8
影: #b39ddb（紫寄りの淡影）
```

**避けること:**
- 子供・人物のリアルな描写
- 街灯の強い光（中央に置かない）
- ボール等は端に小さく配置

**雰囲気キーワード:** 帰り道・甘い予感・公園のざわめき・第2ステージらしい開放感

### 5-3. しょうてんがい（stage_shouten.png）

| 項目 | 値 |
|------|-----|
| 時間帯 | 夕暮れ（提灯がつき始める時間） |
| 場所 | 古い昭和風商店街 |
| キー絵文字 | 🏪 |
| フォールバック色 | `#fff3e0`（薄オレンジ） |

**シーン構成:**
- 上端 30%: 商店街のアーケード屋根・看板の連なり（**文字は読めないモザイク調**）、提灯の連なり
- 中央 40%: アスファルト＋商店街のパース感（消失点を奥に）、薄い夕日の光
- 下端 30%: 左右に商店の入口（八百屋・魚屋・古道具屋等のシルエット）、段ボール箱、自販機

**配色パレット（参考）:**
```
空: #ffccbc → #b39ddb（黄昏グラデ）
看板: #ffab91 / #ffe082 / #f06292（文字は読めない記号化）
アーケード: #8d6e63 / #a1887f
提灯: #ff8a65（小さい点として散りばめる）
道路: #cfd8dc
影: #7e57c2（紫寄りの夕暮れ影）
```

**避けること:**
- 看板に**実在しそうな文字**を書く → 著作権・読みやすさの問題
- 中央に提灯を密集させる
- 群衆や人のシルエット

**雰囲気キーワード:** ノスタルジー・夕暮れ時のゆらぎ・第3ステージの少し緊張感のある雰囲気

---

### 5-4. はいきょ（stage_haikyo.png）

| 項目 | 値 |
|------|-----|
| 時間帯 | 夜・月明かり（暗すぎず幻想的に） |
| 場所 | 廃墟・打ち捨てられた建物の中 |
| キー絵文字 | 🏚️ |
| フォールバック色 | `#efebe9`（薄茶） |

**シーン構成:**
- 上端 25%: 崩れた天井から差し込む月光、星空（**ただし黒すぎない**藍色 `#5c6bc0`）
- 中央 50%: 床に差し込む光のグラデ、ぼかしたガラクタ（フェードしている）
- 下端 25%: 倒れた家具・蔓・草、割れた窓、扉のシルエット

**配色パレット（参考）:**
```
空・月: #5c6bc0 / #9fa8da / #fff59d（月）
壁: #8d6e63 / #a1887f（茶色）
床: #6d4c41 / #d7ccc8（光が当たる部分）
草・蔓: #689f38（彩度抑え）
ガラクタ: #78909c
影: #283593（深い紫紺、黒は使わない）
```

**避けること:**
- 完全な真っ黒（`#000`）→ 不安感が強すぎる、UIの可読性も落ちる
- 暴力的な要素（壊れた血痕・武器など）→ 用語ルール違反
- 怖い表情の影・幽霊
- お化けや骨など

**雰囲気キーワード:** 月夜の静けさ・ノスタルジー・最終ステージの達成感を彩る場所

## 6. ステージ間遷移

ステージ移行時に背景を切り替える際の演出案:

| 演出 | 用途 |
|------|------|
| **クロスフェード** (opacity 0→1, 0.5s) | 通常の `nextStage` 時 |
| **ホワイトイン** (opacity 0, 白フラッシュ後にフェードイン) | クリアパネル「次のステージへ」押下時 |
| **インスタント** | リトライ時（テンポ重視） |

実装は `STAGE_BG_URLS[stageIdx]` を `key` 付きで切り替え、`@keyframes ambientFadeIn 0.5s` を当てる。

## 7. アクセシビリティ・モバイル配慮

### 7-1. パフォーマンス

- 画像合計サイズ: 4 ファイル × 平均 250KB = **1MB 程度**を上限とする
- 初回ロード時にすべて prefetch しない（**現在のステージのみ**を遅延ロード）
- スマホで `backdrop-filter` ＋ 大きな背景画像は GPU 負荷が高い → デバッグオプションで切り替え可能にする計画

### 7-2. 視認性

- 中央 50% 領域の **平均明度 ≥ 88%、コントラスト ≤ WCAG AA 1.5:1** を保つ
  → 盤面の文字（赤数字・緑数字）が必ず読める
- 色覚多様性対応: 赤緑偏重を避ける（背景に強い赤緑を使わない）

### 7-3. データセーブモード

- ユーザー設定で「背景画像をオフ」にできるオプションを将来追加（→ `roadmap.md` に追記）
- オフ時は従来の単色 `bg` ＋ ambient ブロブのみ表示

### 7-4. レスポンシブ

| 画面サイズ | 対応 |
|-----------|------|
| ≤ 480px (スマホ縦) | `background-size: cover; background-position: center top;` |
| 481〜900px (タブレット縦) | 同上 |
| > 900px (PC横/タブレット横) | `background-size: cover; background-position: center center;` |

## 8. 制作ワークフロー

既存アセット（`main.png`, スプライトシート）と揃える:

1. **AI生成（初稿）** — 1500×2250px, アニメ調, ステージ別プロンプト
2. **トリミング・調整** — UIゾーンに干渉する要素を消す（ペイントツール）
3. **量子化** — Pillow で 256色パレット + Floyd-Steinberg ディザリング
4. **オプティマイズ** — `pngquant` または `mozjpeg` で 80% 圧縮
5. **配置** — `src/assets/stages/` に commit
6. **import 追加** — `src/constants/stages.ts` に `bgImage` フィールド追加
7. **動作確認** — 4 ステージとも盤面の数字が読めることをチェック

### 8-1. AIプロンプトテンプレート

各ステージ別の **完全なプロンプト** は別仕様書 [`stage-bg-prompts.md`](./stage-bg-prompts.md) に集約。
英語版（Midjourney / DALL-E 3 用）と日本語版（NovelAI 用）、ネガティブプロンプト、ツール別チューニングを掲載。

簡易テンプレート:

```
"Soft pastel anime illustration, [SCENE], [TIME OF DAY], no characters in the center,
 vignette, painterly, gentle lighting, vertical composition 2:3,
 muted saturation, soft outlines, warm/cool color balance"
```

各ステージ別の `[SCENE]` `[TIME OF DAY]` は §5 を参照。

## 10. チェックリスト（受け入れ基準）

各ステージ画像を追加する際の確認項目:

- [ ] 解像度 1200×1800〜1500×2250、ファイルサイズ ≤ 350KB
- [ ] 中央 50% 領域に細かいディテール・濃い影なし
- [ ] パステル基調、彩度を抑えた配色
- [ ] 動物（猫・犬・鳥）が中央付近に描かれていない
- [ ] 看板に実在する文字が含まれていない（しょうてんがい）
- [ ] 暴力・怖さを煽る要素なし（はいきょ含む）
- [ ] iPhone 実機で UI（数字・スプライト）が読める
- [ ] glass パネル越しに背景の色味が美しく見える
- [ ] フォールバック単色（`STAGES[].bg`）と統一感がある
